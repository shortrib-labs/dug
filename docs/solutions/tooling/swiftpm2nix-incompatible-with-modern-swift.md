# swiftpm2nix incompatible with modern Swift workspace-state.json

## Problem

When running `swiftpm2nix` to generate Nix fetch expressions for a Swift package, the tool fails with:

```
Unsupported .build/workspace-state.json version
```

This occurs because modern Swift toolchains (6.1+) produce `workspace-state.json` version 7, while `swiftpm2nix` in nixpkgs only supports older versions.

## Root Cause

`swiftpm2nix` reads `.build/workspace-state.json` (generated by `swift package resolve`) to extract dependency information and produce Nix fetch expressions. The workspace-state format has evolved across Swift versions, and `swiftpm2nix` has not been updated to parse newer format versions.

This is a known limitation. The nixpkgs Swift documentation recommends swiftpm2nix as the standard approach, but it lags behind the Swift toolchain shipped on modern macOS.

## Solution

Skip swiftpm2nix entirely and fetch dependencies directly using `fetchFromGitHub` in the Nix derivation. This is consistent with how many nixpkgs packages handle Swift dependencies when swiftpm2nix is unavailable or impractical.

Instead of:
```nix
# swiftpm2nix approach (broken)
let generated = swiftpm2nix.helpers ./nix;
in stdenv.mkDerivation {
  configurePhase = generated.configure;
  # ...
}
```

Use direct fetching with a synthetic workspace-state.json in v5 format (compatible with nixpkgs Swift 5.10.1):

```nix
let
  dep-src = pkgs.fetchFromGitHub {
    owner = "apple";
    repo = "swift-argument-parser";
    rev = "626b5b7b2f45e1b0b1c6f4a309296d1d21d7311b";
    hash = "sha256-90ECc3iEmxvOUk9iLKbQdQEz88dOisPqWsJLOFcKUV8=";
  };

  workspaceState = builtins.toFile "workspace-state.json" (builtins.toJSON {
    version = 5;  # Must match what nixpkgs Swift understands
    object = {
      artifacts = [];
      dependencies = [{
        basedOn = null;
        packageRef = {
          identity = "swift-argument-parser";
          kind = "remoteSourceControl";
          location = "https://github.com/apple/swift-argument-parser";
          name = "swift-argument-parser";
        };
        state = {
          checkoutState = { revision = "626b5b7b..."; version = "1.7.1"; };
          name = "checkout";
        };
        subpath = "swift-argument-parser";
      }];
    };
  });
in stdenv.mkDerivation {
  configurePhase = ''
    mkdir -p .build/checkouts
    install -m 0600 ${workspaceState} .build/workspace-state.json
    ln -s ${dep-src} .build/checkouts/swift-argument-parser
  '';
  buildPhase = ''
    swift build --disable-sandbox -c release
  '';
}
```

The dependency information comes from `Package.resolved` (version 2 format). The workspace-state version must match the nixpkgs-provided Swift compiler (v5 for Swift 5.10.1), not the version your development Swift generates (v7 for Swift 6.1+). See `nix/package.nix` for the complete working derivation and `flake.nix` for the flake wrapper.

## Prevention

- When planning Nix packaging for Swift projects, check swiftpm2nix compatibility early by running it against the project's workspace-state.json before committing to the swiftpm2nix approach.
- Monitor nixpkgs Swift packaging discussions for updates to swiftpm2nix or alternative tooling.
- The `Package.resolved` format version (currently 2) is separate from the `workspace-state.json` version (currently 7). The resolved file is stable and human-readable; prefer it as the source of truth for dependency information.
